Inspect the repository before making any changes.

Tasks:
1. list the current directory tree at a practical depth
2. identify whether this is already a Python project
3. inspect for `pyproject.toml`, `requirements.txt`, `poetry.lock`, `.env.example`, `README.md`, `tests/`, and existing app structure
4. check Python version
5. identify missing project foundations
6. summarize the current state before changing anything

Output:
- current structure summary
- important existing files
- detected tooling
- missing foundations
- recommended first implementation step
